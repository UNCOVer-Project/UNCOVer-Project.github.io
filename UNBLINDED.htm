UNBLINDED - From Sightless to Sightful

Background

A sightless person has difficulties in indentifying objects. But, it doesn't mean that a sightless person cannot identify and remember objects because their other senses work better than most of normal people as a natural compensation due to their deficiency. They can sense their surroundings and know the presence of objects with their heightened perception.

But, with all of their adaptation, there is still one differentiating factor that makes them less precise in detecting object, that is finding and locating small sized and complex shaped object.

In general scenario, a sightless person is very dependant in memory and their senses. They can quickly identify objects that are routinely used and remember, but may have difficulties indentifying completely new object and/or in different/unusual locations.

As an example, there is a spoon, fork, and other utensils and foods above a table. If the sightless person has no other person to assist, or the place is completely new to the person, he/she will have difficulties to eat with proper utensils and food. It is also bad for them to touch those utensils that may have been contaminated from touching with their hand.

Based on this problem, we offer a solution that we named "UNBLINDED".

What is Unblinded

UNBLINDED is an accessibility tool, shaped like a sunglasses that is used as a headwear. UNBLINDED is intended for sightless person to help them identify objects in direction they are facing to. UNBLINDED works by identifying the object that are captured in video with computer vision and then speaks the objects' name and location to the user.

UNBLINDED core component consists of a microphone to record user's voice command, a speaker to speak the object information to the user, a camera to capture short video in the direction of the user, and a microcontroller to process and manage the data.

How It Works

While the device is on, the microphone will constantly monitor for user's speech. The user can speak "What's in front of me" in order for UNBLINDED to start capturing footage for three seconds. After that, the footage file will be sent to an Computer Vision API to be analyzed for individual objects. The API will then return information of object names and locoations. If there are no object detected, then text-to-speech engine will output "There are no detected objects in front of you" through the speaker. If there are object(s) detected, then the device will be speaking each object(s) name and location. Alternatively, if a finger pointing to a specific object is detected, then the device will speak the pointed object's name. After that, the microphone will continue to ckemonitor user commands. If the user say "Turn off", then the device will stop monitoring and turn off. Else, the device will keep monitoring and work as described.

Project's Pillars

In this section we will the describe the 'Pillars' of this program. So, here it is.

Raspbian OS

Raspbian OS is the default OS of Raspberry Pi, Raspbian is basically based on linux, so we can run our script more or less just like on our PC.

Python

We use Python for simplicity, versatility, and cross-platform compatibility. Also, Azure services supports Python. So, yeah.

Azure Vision Cognitive Services

We use Azure Vision Cognitive Services for analyzing images we captured from the camera. With Azure's Vision and Custom Vision APIs, we can determine what objects are in front of the sightless person. It also gives the location of the item in the picture.

Azure Speech Cognitive Services

Using Azure's Speech Cognitive Services, we can translate user commands into text using Speech-to-Text service, so we can process user's request accordingly. The Text-To-Speech service is used to convert image analyze result (from a string), to a voice (or a speech) which later will be played to the user.

Raspberry Pi 3 Model B

Raspberry Pi 3 Model B has everything we need for this project, which is a wi-fi module, 3.5 mm jack, USB Ports, and a lite Linux Distro made by Raspberry to run our program scripts. Raspberry Pi 3 Model B is fast enough to do everything we need.

USB Microphone

This USB Microphone is plug and play on Raspberry Pi. It's used to listen for user's commands.

Battery

We're designing a wearable tech. So it should be mobile, and it means battery. This battery and its controller are specifically designed for use with Raspberry 3 Model B.

Camera

The camera is used to take images for the software. From the picture of this camera, later it'll be processed using Azure Vision Cognitive Services and later on described through voice for the user.

Earphone

Rather than using speakers, we decided to use Earphone for the user to listen the output voice of the program. It helps voice insulation, the volume is adjustable, can be removed when not used, and cost-efficient. It's also applicable as an alternative for microphone.

Sunglasses

Sunglasses is a common thing that sightless people use. So definitely we use it for this project.
